XAML入门

XMAL（Extensible Application Markup Language）可扩展程序标记语言是微软开发的一种基于 XML 、声明式的界面描述语言。XAML 与C#、VB.NET 同属 .NET 平台，可被编译成 IL 代码。

声明式即以标签的方式声明元素，编译器将自动创建对应的 CLR 对象，等价于调用了某个类的构造函数。

命名空间

为了避免命名冲突，XAML 沿用了 XML 中命名空间的概念。命名空间只能在根元素中声明，根元素必须引入至少一个命名空间，用于验证根元素本身及其子元素。

引入命名空间

命名空间引入语法为xmlns:空间名称="xxx"，括号右侧分两种情况：

  1. 引入某类库中的命名空间

    xmlns:空间名称=”clr-namesapce:类库中的空间名称;assembly=类库本身文件名”。比如要引用mscorlib类库中的System命名空间，写法如下：

    xmlns:sys = "clr-namespace:System;assembly=mscorlib"

  2. URL

    对于每个 WPF 程序都依赖的命名空间，逐个引入非常麻烦。为了简化代码，XAML 中采用不同 URL 地址分别代表它们。

    xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
xmlns:mc="http://schemas.openxmlformats.org/markup-compatibility/2006" 
xmlns:d="http://schemas.microsoft.com/expression/blend/2008"

默认命名空间

默认命名空间的名称可以省略，比如上节中的xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"。声明命名空间下的元素时也无需指定空间前缀，比如各类控件的声明。

x命名空间

x命名空间与解析编译 XMAL 代码相关，其下属几个常用特性如下：

  • x:Name

    用于指定元素名称

  • x:Class

    此特性用于根元素，告诉编译器与后台哪个partial类进行合并。新建窗口、页面及控件时，Visual Studio 会自动生成前台 XMAL 代码和后台 C# 代码，但后者非必须。

  • x:key

    WPF 资源是以“key-value”形式存储的，通过 key 识别到某个具体资源。

  • x:static

    在前台代码中访问后台类的静态成员。在多国语言程序中，一般将不同语言的字符串保存到后台资源类的静态属性中。

    <TextBlock Text="{x:Static Local:I8n.ENG_AGE}"

元素

元素通常由一对开始标签和结束标签组成。标签体包含内容的标签称为非空标签，不包含内容的标签称为空标签，两者写法如下。

<!--非空标签-->
<StackPanel> 
   <Button />  <!--空标签-->
</StackPanel>

元素特性

特性为元素附加了额外的信息，在 WPF 中与类的属性或事件是同一事物的两种不同表现形式，但并不完全对应。特性有两种赋值方式：

  • 简单赋值：使用字符串
  • 复杂赋值：使用特性元素

简单赋值采用Attribute=“Value”语法，示例如下。

<TextBlock Text="用户名："></TextBlock>

特性元素

元素的某个特性以元素的形式出现，表现形式是“类型名.属性名(TypeName.PropertyName)”，比如下例中Window元素的DataContext特性，并且总会声明在所属元素的标签体中。特性元素本质还是特性，因此没有属于自己的特性。

<Window
    x:Class="WpfApp3.MainWindow"
    xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
    xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
    xmlns:d="http://schemas.microsoft.com/expression/blend/2008"
    xmlns:mc="http://schemas.openxmlformats.org/markup-compatibility/2006"
    Title="MainWindow"
    Width="800"
    Height="450"
    mc:Ignorable="d">

    <Window.DataContext>
        <vm:ViewModelMain />
    </Window.DataContext>
    
</Window>

标记扩展

标记扩展延伸了特性的取值范围，简单赋值不再只涉及字符串值，也可以是静态资源、对象等等。只要特性值被一对花括号括起来，XAML 编译器就认为是标记扩展值而不是普通的字符串值。标记扩展支持嵌套。

标记扩展语法为Attribute=“{xxx}”，花括号中的第一个标志符是扩展类的类名，扩展类派生于MarkupExtension类。标记扩展类的名称按惯例以 Extension 后缀结尾，但 XAML 中可省略该后缀。

标记扩展可分为 XAML 标扩展和 WPF 扩展两大类，常用 XAML 标记扩展如下：

  • x:Null

    空标记扩展，特性值设为 NULL。常用于放弃相关特性的继承。

    <Window.Resources>
    <Style TargetType="Button">
        <Setter>
            <Setter.Property>Button.Background</Setter.Property>
            <Setter.Value>SeaGreen</Setter.Value>
        </Setter>
    </Style>
</Window.Resources>

<Button Style="{x:Null}">Click Me</Button>

  • x:Static

    静态属性扩展，在 XAML 中访问静态成员。

  • x:Type

    对象类型扩展，在 XAML 中标记数据类型。

  • x:Array

    静态数组扩展，在 XAML 中创建静态数组。

    <Window.Resources>
    <x:Array x:Key="boys" Type="{x:Type sys:String}">
        <sys:String>猴子</sys:String>
        <sys:String>肉肉</sys:String>
        <sys:String>二哈</sys:String>
    </x:Array>
</Window.Resources>
<StackPanel>
    <ListBox ItemsSource="{StaticResource boys}"></ListBox>
</StackPanel>

类型转换器

类型转换器用于将字符串字面量转换为相应的数据类型，WPF 本身内置了常用类型的转换器，比如Brush、Color、FontWeight、Point等，它们都派生自TypeConverter类。XMAL 语言本身区分大小写，转换器不区分字符串的大小写。

绑定

  • Binding

    数据绑定扩展，将自身的某个特性与任意数据绑定，MVVM中频繁用到此标记扩展。

  • TemplateBinding

    为了方便复用，可以将控件的样式设置集中到一个控件模板里，然后通过模板绑定的方式应用到具体控件。

    <Application.Resources>
    <ControlTemplate x:Key="RedButton" TargetType="Button">
        <ContentControl Content="Click" Foreground="Red"></ContentControl>
    </ControlTemplate>
</Application.Resources>

<Button Template="{StaticResource RedButton}"></Button>

资源

通常所谓的资源（二进制资源）指程序代码用到的一些附件内容，比如图标、版权字符串等。为了防止遗失或被破坏，可将资源编译进程序的主体，与程序主体浑然一体。WPF 另外引入了对象资源的概念，每个元素都可携带能被共享的对象资源，比如样式、控件模板、动画组件及画刷等。对象资源在元素的 Resources 特性元素中定义，以“键—值”对的形式存储。

在检索资源时，先查找当前元素的 Resources 特性，如果没有找到再沿着逻辑树去父元素查找，最终直到Application.Resources 元素。如果搜索完所有路径都没发现，就会抛出异常。

根据使用资源的方式不同，分静态绑定资源和动态绑定资源。绑定资源通过前面提到的标记扩展语法实现。

静态绑定

当程序初始化时一次性地使用某资源，后续不再关注该资源可能的变化时，使用静态绑定扩展标记。

<Window.Resources>
    <sys:String x:Key="motto">Good Good Study,Day Day Up</sys:String>
</Window.Resources>

<TextBlock Text="{StaticResource motto}"></TextBlock>

动态绑定

资源在程序运行后可能发生变化，且被绑定的特性需同步响应变化时，使用动态绑定扩展标记。

<Window.Resources>
    <sys:String x:Key="motto">Good Good Study,Day Day Up</sys:String>
</Window.Resources>

<TextBlock Text="{DynamicResource motto}"></TextBlock>

// 后台改变资源
private void Button_Click(object sender, RoutedEventArgs e)
{
    Resources["motto"] = "so boring";
}

资源文件

为了实现工程化及便于复用，可将资源定义在一个单独的资源字典文件中。该文件类似于前端常见的 CSS 样式文件，后缀名为 .xaml。大家熟悉的 MaterialDesign 界面库就是以资源文件的方式提供的，其引入方式如下：

<Application.Resources>
    <ResourceDictionary Source="pack://application:,,,/MaterialDesignThemes.Wpf;component/Themes/MaterialDesignTheme.Defaults.xaml" />
</Application.Resources>

使用示例代码如下：

<Button Style="{DynamicResource MaterialDesignFlatMidBgButton}">Start</Button>

常用属性

height&width

  • Auto表示自动适应显示内容的宽度或高度.

  • *表示按比例分配，1*同*，为默认值。

<Grid.ColumnDefinitions>
    <ColumnDefinition Width="Auto" />  <!-- 自适应，为所有子元素占据宽度之和 -->
    <ColumnDefinition Width="50.5" />  <!-- 固定值 -->
    <ColumnDefinition Width="50*" />   <!-- 比例，占据剩余宽度的80% -->
    <ColumnDefinition Width="40*"/>    <!-- 比例，占据剩余宽度的10% -->
    <ColumnDefinition Width="*"/>      <!-- 比例，占据剩余宽度的1% -->
    <ColumnDefinition/>      		   <!-- 比例，占据剩余宽度的1% -->
</Grid.ColumnDefinitions>

margin&padding

外边距即元素的外边框到四周相邻元素间的距离，该值允许叠加。比如下图元素间的 margin 均为10px，但实际距离是20px。

外边距有3种表达方式：

Margin="left,top,right,bottom"
Margin="left and right,top and bottom"
Margin="all directions"

内边距指元素内中的子元素（比如按钮中的文字）距四周当前元素内边框的距离，表达方式同外边距。
